## 6.0 社群策略概覽

本系統的核心假設：**好的工具加上好的社群，才能產生真正的社會影響力。**

工具設計了三個社群觸及機制：

### 機制 1：降低發現成本

資料已整理好，使用者只需要「知道去找」。
行動號召：讓公民記者、社運人士、里民知道這個工具存在。

### 機制 2：降低回報成本

Google Forms 預填機制讓「回報一筆」只需要 30-60 秒。
與傳統需要寫信或打電話的監督管道相比，門檻極低。

### 機制 3：累積可見性

每一筆回報都增加了調查資料庫的價值。
當回報累積到一定數量，媒體引用的可能性大幅提升。
媒體引用 → 更多人知道 → 更多回報 → 正向循環。

## 6.1 目標社群

| 社群 | 接觸管道 | 訊息重點 |
|------|---------|---------|
| 公民記者、部落客 | 公民報導者網路、g0v.tw | 調查工具、資料可引用 |
| 大學新聞系 | 系所助教、教授 | 課程作業、調查報導訓練 |
| 里民大會、社區組織 | Facebook 社群、LINE 群組 | 查查你選區的議員金主 |
| 環境、都計倡議 | 地球公民、台灣健康空氣行動聯盟等 | 查特定產業的捐款紀錄 |
| 調查報導媒體 | 報導者、READr 等 | 提供可直接引用的結構化資料 |

## 6.2 開源協作邀請

本專案是開源的（MIT 授權）。
以下是適合新貢獻者的入門任務（適合在 GitHub Issues 標記為 `good first issue`）：

- 新增更多選舉年度的資料轉換
- 改進搜尋效能（如引入 Fuse.js 模糊搜尋）
- 設計更好的視覺化（如 D3.js 關聯圖）
- 翻譯成英文或其他語言
- 撰寫使用說明影片腳本
