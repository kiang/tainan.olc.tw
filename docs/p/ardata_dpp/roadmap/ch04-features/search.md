# 搜尋功能設計規格

---

## 搜尋範圍

使用者可搜尋以下欄位：

| 搜尋內容 | 對應資料欄位 |
|---------|-------------|
| 公司名稱 | business.csv 中的企業名稱 |
| 統一編號 | tax_id（8碼數字） |
| 候選人姓名 | 各公司 CSV 中的「捐贈對象」欄位 |

---

## 行為規格

### Debounce

搜尋輸入框使用 300ms debounce：使用者停止打字 300ms 後才觸發篩選，避免每個按鍵都重新掃描資料。

### Client-side 篩選

所有搜尋在瀏覽器端執行。business.csv（~29K 行）在頁面載入時已全部讀入記憶體，搜尋只是對這份記憶體資料進行 `filter()`，不需要任何 API 呼叫。

### URL 同步

每次搜尋後更新 URL 參數（`?q=關鍵字`），使搜尋結果可以分享。使用者複製 URL 並在新分頁開啟時，會自動套用相同的搜尋條件。

### 搜尋後的資料詳細層

使用者搜尋並找到特定企業後，點擊該企業可展開詳細捐款歷史。此時才觸發 `fetch("report/incomes/business/{taxID}.csv")` 載入該企業的歷次捐款紀錄。

---

## 視覺回饋

- 搜尋時顯示「掃描資料庫中...」打字機效果（可選，參見 gamification.md）
- 結果數量即時更新：「共找到 X 家企業」
- 無結果時顯示友善提示

---

## Tab 切換

頁面有兩個 Tab：

| Tab | 資料來源 | 說明 |
|-----|---------|------|
| 企業捐款 | business.csv | 按企業總金額排序 |
| 個人捐款 | individual.csv | 按個人總金額排序 |

切換到「個人捐款」Tab 時才載入 individual.csv（延遲載入）。URL 參數 `?tab=business` 或 `?tab=individual` 記錄當前 Tab。
