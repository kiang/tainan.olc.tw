# 台南凸面鏡地圖 - 專案計畫書

## 專案概述

建立一個群眾外包的凸面鏡（交通反射鏡）狀態追蹤平台，讓市民能夠回報和追蹤全台灣凸面鏡的設置位置與維護狀態。凸面鏡是重要的交通安全設施，可能因損壞、遮蔽或缺失而影響用路人安全。

## 架構設計

本專案採用與 `youbike`（YouBike 站點地圖）相似的純前端架構，使用 Leaflet 作為地圖引擎，涵蓋全台灣：

- **地圖引擎**: Leaflet 1.9.4 + Leaflet.markercluster 1.5.3
- **UI 框架**: Bootstrap 5.3.0 + Bootstrap Icons
- **圖表**: Chart.js
- **CSV 解析**: PapaParse 5.4.1
- **資料來源**: Google Sheets（發布為 CSV）
- **回報機制**: Google Forms（預填連結）
- **流量分析**: Google Analytics
- **底圖**: 國土測繪圖資服務雲 WMTS

## 資料模型

每筆凸面鏡回報包含以下欄位：

| 欄位 | CSV 欄位索引 | 說明 |
|------|-------------|------|
| timestamp | 0 | 提交時間（YYYY-MM-DD HH:MM:SS） |
| fileUrl | 1 | 照片網址（Google Drive 連結） |
| status | 2 | 狀態：正常 / 損壞 / 遮蔽 / 缺失 |
| city | 3 | 縣市名稱 |
| town | 4 | 鄉鎮市區名稱 |
| longitude | 5 | 經度座標 |
| latitude | 6 | 緯度座標 |
| uuid | 7 | 唯一識別碼（UUID v4） |
| hasLocal | 8 | 是否有本地圖片（0/1） |
| address | 9 | 街道地址或地標描述 |
| notes | 10 | 凸面鏡狀態的額外備註 |
| reply | 14 | 政府／管理單位回覆 |

## 狀態分類與配色

| 狀態 | 顏色代碼 | 說明 |
|------|---------|------|
| 正常 | `#4CAF50`（綠色） | 凸面鏡完好且功能正常 |
| 損壞 | `#F44336`（紅色） | 凸面鏡破裂、變形或脫落 |
| 遮蔽 | `#FF9800`（橙色） | 被植物、招牌等遮擋視線 |
| 缺失 | `#9E9E9E`（灰色） | 凸面鏡已不存在（僅剩支架或完全消失） |

## 功能規劃

### 核心功能

1. **互動式地圖** — 使用 Leaflet 搭配 WMTS 底圖（國土測繪圖資服務雲），初始視角涵蓋全台灣
2. **群集標記** — 使用 Leaflet.markercluster，依狀態著色（綠=正常、紅=損壞、橙=遮蔽、灰=缺失）
3. **彈出詳情** — 顯示照片、狀態、地址、備註、時間、政府回覆
4. **篩選搜尋** — 依狀態分類篩選 + 地址文字搜尋
5. **GPS 定位** — 定位按鈕取得使用者目前位置
6. **座標輸入** — 手動輸入經緯度跳轉
7. **打卡模式** — 現場巡檢驗證凸面鏡狀態
8. **資料提交** — Google Forms 連結回報新凸面鏡或新增照片
9. **統計圖表** — 長條圖顯示各狀態數量統計
10. **URL 路由** — 基於 hash 的深層連結（#point/:uuid）
11. **導航連結** — 路線規劃（Google Maps、Bing Maps、HERE Maps）

### 使用者互動流程

```
使用者開啟地圖
    ↓
瀏覽凸面鏡標記（依狀態著色）
    ↓
點擊標記 → 查看詳細資訊（照片、狀態、地址）
    ↓
可選擇：
  - 新增照片（連結至 Google Form）
  - 導航前往（Google/Bing/HERE Maps）
  - 查看處理情形（政府回覆）
    ↓
點擊空白處 → 回報新凸面鏡（連結至 Google Form，自動帶入座標）
```

### 打卡模式流程

```
啟動打卡模式
    ↓
前往凸面鏡位置
    ↓
點擊標記完成打卡（標記變色）
    ↓
結束打卡 → 顯示統計（已打卡數量、耗時）
```

## 資料流程

```
Google Sheets（主資料）
    ↓ PapaParse 下載並解析 CSV

```

## Google Forms AI 生成提示

### 表單一：凸面鏡回報表單（主表單）

請將以下提示貼入 Google Forms 的「使用 AI 協助建立表單」功能：

> 建立一個中文表單，標題為「凸面鏡回報」，描述為「回報全台灣凸面鏡（交通反射鏡）的位置與狀態，協助追蹤維護情形」。表單需包含以下欄位，請依照順序建立：
>
> 1. 「照片」— 檔案上傳欄位，必填，說明文字為「請上傳凸面鏡的現場照片」
> 2. 「狀態」— 下拉選單，必填，選項為：正常、損壞、遮蔽、缺失。說明文字為「正常＝完好可用；損壞＝破裂變形脫落；遮蔽＝被植物招牌等擋住；缺失＝已不存在」
> 3. 「縣市」— 簡答文字，必填
> 4. 「鄉鎮市區」— 簡答文字，必填
> 5. 「經度」— 簡答文字，必填，說明文字為「由地圖自動帶入」
> 6. 「緯度」— 簡答文字，必填，說明文字為「由地圖自動帶入」
> 7. 「UUID」— 簡答文字，必填，說明文字為「由系統自動產生，請勿修改」
> 8. 「是否有本地圖片」— 簡答文字，非必填，預設值為 0
> 9. 「地址」— 簡答文字，非必填，說明文字為「街道地址或附近地標」
> 10. 「備註」— 長文字（段落），非必填，說明文字為「其他補充說明，例如損壞程度、遮蔽原因等」
>
> 表單設定：收集電子郵件地址設為否，限制為每人一次回覆設為否（允許多次提交），回覆的試算表自動建立。

**建立後設定：**
- 進入「回覆」分頁 → 連結至 Google Sheets
- 在 Google Sheets 中：檔案 → 共用 → 發布到網路 → 選擇 CSV 格式 → 發布
- 複製 CSV 發布網址，填入 `main.js` 的 `DATA_CSV_URL`
- 複製表單的預填連結網址，填入 `main.js` 的 `REPORT_FORM_URL`
- 預填連結的 entry ID 對應：
  - `entry.1588782081` → 縣市
  - `entry.1966779823` → 鄉鎮市區
  - `entry.1998738256` → 經度
  - `entry.1387778236` → 緯度
  - `entry.2072773208` → UUID

  - `entry.1998738256` → 經度
  - `entry.1387778236` → 緯度

**重要提醒：** 建立表單後，實際的 entry ID 會由 Google Forms 自動產生，與上方列出的 ID 不同。請在建立表單後取得實際的預填連結，並更新 `main.js` 中對應的 entry ID。取得方式：在表單編輯頁面點選右上角「⋮」→「取得預先填入的連結」→ 填入範例值 → 取得連結 → 從 URL 中擷取各欄位的 entry ID。
