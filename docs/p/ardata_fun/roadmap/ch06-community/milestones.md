# 開發里程碑：Phase 1-4

---

## 里程碑概覽

```
Phase 1 — 基礎建設（目標：1 週）
  從零到有第一個可用版本

Phase 2 — 功能完善（目標：2 週）
  從可用到好用

Phase 3 — 趣味化與社群（目標：3 週後）
  從好用到讓人想推薦給別人

Phase 4 — 社群推廣與永續（持續進行）
  從工具到公民運動
```

---

## Phase 1：基礎建設

**目標**：建立最小可行產品（MVP），能讓使用者搜尋資料並回報

**預計時程**：7 天（1 名開發者全職 or 2 名開發者兼職）

### 任務清單

**Day 1-2：資料確認**

| 任務 | 預估時間 | 完成標準 |
|------|---------|---------|
| 確認 kiang.github.io/ardata.cy.gov.tw CSV 可正常 fetch | 0.5h | `curl ... | head -5` 回傳正確資料 |
| PapaParse 解析測試：business.csv 可正確解析 | 1h | 解析後 total_amount 為數字，tax_id 為 8 碼字串 |
| 確認個別企業 CSV 格式（business/{taxID}.csv） | 0.5h | 標頭正確，日期欄位可解析 |

**Day 3-4：表單**

| 任務 | 預估時間 | 完成標準 |
|------|---------|---------|
| 建立 Google 表單，新增所有欄位 | 1h | 8 個欄位全部建立 |
| 取得所有欄位的 Entry ID | 0.5h | 3 個預填欄位的 entry.ID 已記錄 |
| 連結 Google Sheets，設定通知 | 0.5h | 測試提交後 Sheets 有記錄 |
| 送出 3 筆測試回報 | 0.5h | 預填資料正確，Sheets 有記錄 |

**Day 4-5：前端（核心）**

| 任務 | 預估時間 | 完成標準 |
|------|---------|---------|
| 建立 index.html 基本結構 | 2h | 可在瀏覽器開啟，版面不破板 |
| 實作 business.csv 載入與顯示 | 2h | 排行榜表格可顯示企業名稱與總金額 |
| 實作搜尋（公司名稱與統編） | 2h | 搜尋關鍵字後表格即時更新 |
| 實作企業 CSV 展開（點擊查看詳細） | 2h | 點擊企業後載入歷次捐款明細 |

**Day 5-6：整合回報功能**

| 任務 | 預估時間 | 完成標準 |
|------|---------|---------|
| 填入 FORM_BASE_URL 和 ENTRY IDs | 0.5h | 設定值不含 YOUR_FORM_ID |
| 實作 handleReport() | 1h | 點擊回報後開啟有預填資料的表單 |
| 測試 10 筆真實記錄的回報流程 | 1h | 10 筆都成功，Sheets 有記錄 |
| 加入法律免責聲明橫幅 | 0.5h | 橫幅顯示且可關閉 |

**Day 7：部署**

| 任務 | 預估時間 | 完成標準 |
|------|---------|---------|
| 推送 index.html 到 GitHub | 0.5h | git push 成功 |
| 確認 GitHub Pages 可存取 | 0.5h | 網址可開啟，功能正常 |
| 跑完上線前清單的 A+B+C 項目 | 2h | 所有 (必要) 項目打勾 |
| 找 3 人進行用戶測試 | 1h | 收到回饋，無重大 bug |

### Phase 1 驗收標準

- [ ] 頁面載入後顯示企業捐款排行榜
- [ ] 使用者可搜尋公司名稱或統一編號
- [ ] 點擊企業可展開歷次捐款明細
- [ ] 每筆記錄有「回報關聯案件」按鈕
- [ ] 點擊回報後開啟 Google 表單，捐款者等欄位已帶入
- [ ] 法律聲明橫幅存在且文字正確
- [ ] 在手機（375px 寬）上可以完成一次完整的搜尋 + 回報操作

## Phase 2：功能完善

**目標**：讓系統從「能用」到「好用」

**預計時程**：Phase 1 結束後 2 週

### 任務清單

| 任務 | 預估時間 | 優先順序 | 完成標準 |
|------|---------|---------|---------|
| 個人捐款 Tab（individual.csv） | 3h | P1 | 切換 Tab 可載入個人排行榜 |
| 表格排序（金額欄位） | 2h | P1 | 點擊表頭可排序 |
| 分頁功能（50筆/頁） | 3h | P1 | 超過50筆時分頁按鈕出現 |
| URL 參數同步（q, tab） | 2h | P1 | 複製 URL 在另一分頁可還原狀態 |
| 行動裝置優化（表格水平捲動） | 2h | P1 | iPhone SE 上不破版 |
| og_image.png 製作 | 2h | P1 | 社群分享時顯示自訂預覽圖 |
| 加入資料來源連結 | 0.5h | P1 | 頁腳有指向 ardata.cy.gov.tw 的連結 |
| 廉政署檢舉熱線資訊 | 0.5h | P1 | 頁腳顯示 0800-286-586 |

### Phase 2 驗收標準

- [ ] 所有 P1 功能均已實作並通過手動測試
- [ ] URL 參數分享功能正常
- [ ] Lighthouse Performance 分數 > 60
- [ ] 至少 5 位測試者試用後無重大 bug 回報

## Phase 3：趣味化與社群觸及

**目標**：讓系統從「有用」變成「有趣且讓人想分享」

**預計時程**：Phase 2 結束後 3 週

### 任務清單

**偵探敘事框架**

| 任務 | 預估時間 | 完成標準 |
|------|---------|---------|
| 首次訪問 Onboarding 文案 | 2h | 偵探主題的歡迎訊息 |
| 頁面頂部進度顯示（查閱 X 家企業） | 2h | 即時更新已查閱數量 |
| 偵探職階系統 | 3h | 6 個職階可晉升，有晉升特效 |

**成就系統**

| 任務 | 預估時間 | 完成標準 |
|------|---------|---------|
| 實作 Achievements 模組（localStorage） | 4h | 第一次點擊回報後顯示「第一份報告」成就 |
| 設計 10 個成就的圖示與說明文字 | 2h | 所有成就在 UI 中顯示 |
| 成就 Toast 通知 + confetti 動畫 | 2h | 解鎖時有動畫效果 |
| 大額捐款脈衝 glow 動畫（>100萬） | 1h | 大額記錄有紅色閃爍效果 |

**社群功能**

| 任務 | 預估時間 | 完成標準 |
|------|---------|---------|
| 企業詳頁「分享此調查」按鈕 | 2h | 點擊後複製含統一編號的 URL |
| 「今日偵探報告」面板 | 2h | 顯示當天查閱過的企業 |

### Phase 3 驗收標準

- [ ] 成就系統正常：10 個成就可以解鎖
- [ ] 第一次使用者回報後，3 秒內出現成就 Toast
- [ ] 至少 10 人使用後收到 > 20 筆有效回報
- [ ] 職階晉升特效正常運作

## Phase 4：社群推廣與永續

**目標**：讓系統被正確受眾發現並持續使用

**預計時程**：持續進行

### 推廣策略

| 任務 | 時間 | 目標受眾 |
|------|------|---------|
| 寫給調查媒體的介紹信（報導者、READr） | 2h | 專業記者 |
| 投稿 g0v.hackmd 社群 | 1h | 公民科技社群 |
| 製作 1 分鐘使用示範影片 | 4h | 一般大眾 |
| 在相關 Facebook 社群分享 | 0.5h | 關心政治的公民 |

### 資料維護

| 任務 | 頻率 |
|------|------|
| 新選舉後更新 PHP Pipeline 資料 | 每次選舉 |
| 確認 Google Forms 回覆正常運作 | 每月 |
| 審核 Google Sheets 中的新回報 | 每週 |

### Phase 4 長期願景

- 至少 2 個縣市的地方公民媒體持續使用本系統
- 至少 1 篇完整調查報導引用本系統的回報資料
- 系統資料涵蓋至少 4 個選舉年度
