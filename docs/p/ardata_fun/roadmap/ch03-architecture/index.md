## 架構設計哲學

### 純前端、零後端

本系統刻意選擇純前端架構，理由如下：

**可複製性**：任何人 fork 後，不需要設定伺服器、資料庫、環境變數，五分鐘即可運行自己的版本。這對公民科技專案的擴散至關重要。

**可稽核性**：所有程式碼均在 GitHub 公開，任何人都可以審查資料處理邏輯，確認沒有在後端偷偷修改資料。

**零維護成本**：GitHub Pages 免費、穩定，無需管理伺服器、憑證、資料庫備份。

**資料透明**：JSON 資料檔案也是 git 版本控制的一部分，任何資料修改都有記錄。

## 技術選型決策記錄

| 技術 | 選擇 | 理由 | 捨棄的替代方案 |
|------|------|------|--------------|
| CSS 框架 | TailwindCSS (CDN) | 無需建構工具、快速原型 | Bootstrap（過重）、手寫 CSS（難維護） |
| JS 框架 | Vanilla JS (ES6+) | 零依賴、無 bundle 步驟 | React/Vue（需 Node.js 建構環境） |
| 資料存儲 | JSON 靜態檔案 | 直接被瀏覽器 fetch，GitHub Pages 友善 | SQLite-wasm（過複雜）、localStorage（大小限制） |
| 回報機制 | Google Forms 預填 URL | 免費、無需後端、Google Sheets 自動匯整 | Typeform（付費）、自建表單（需後端） |
| 成就存儲 | localStorage | 無需帳號系統，即時回饋 | IndexedDB（過複雜）、後端資料庫（需帳號） |
| 地圖/可視化 | 保留彈性（建議 D3.js） | 按需引入 | ECharts（過重） |
| 部署 | GitHub Pages | 免費、與程式碼同倉庫 | Vercel（需帳號整合）、Netlify |
