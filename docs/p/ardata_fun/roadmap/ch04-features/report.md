# 回報流程設計規格

---

## 設計原則

1. **預填自動**：使用者點擊「回報關聯案件」時，Google 表單的三個系統欄位已自動帶入捐款資訊，使用者只需補充「我知道的那部分」
2. **開新分頁**：回報表單在新分頁開啟，使用者填寫完畢後關閉，原頁面狀態不受影響
3. **成就觸發**：使用者從 Google Forms 新分頁返回後，系統偵測並觸發相關成就（FIRST_REPORT 等）
4. **Toast 確認**：表單開啟後在原頁面顯示短暫的 Toast 通知「已開啟回報表單」

---

## Google Forms 欄位對應表

| 表單欄位 | 資料來源 | 自動填入？ |
|---------|---------|-----------|
| 捐款者名稱與統編 | 企業名稱 + tax_id | 是 |
| 捐款日期與金額 | 捐贈日期 + 捐贈金額 | 是 |
| 收受候選人資訊 | 捐贈對象 + 選舉名稱 | 是 |
| 關聯標案/都計/政策名稱 | — | 使用者填寫 |
| 案件發生或決標日期 | — | 使用者填寫（選填） |
| 公開資訊連結 | — | 使用者填寫（選填） |
| 對價關係說明 | — | 使用者填寫（必填） |
| 聯絡方式 | — | 使用者填寫（選填） |

---

## 預填 URL 格式

```
https://docs.google.com/forms/d/e/{FORM_ID}/viewform?usp=pp_url
  &entry.111111111={捐款者名稱與統編}
  &entry.222222222={捐款日期與金額}
  &entry.333333333={候選人與選舉資訊}
```

URL 參數使用 `URLSearchParams` 自動處理 URL 編碼（含中文、特殊字元）。

---

## 彈出視窗被攔截時的備用方案

若瀏覽器攔截 `window.open()`：
1. 嘗試用 `navigator.clipboard.writeText()` 複製表單 URL
2. 顯示 Toast：「表單連結已複製到剪貼簿，請手動貼上開啟」
3. 若剪貼簿也不可用，用 `window.prompt()` 顯示連結讓使用者手動複製
