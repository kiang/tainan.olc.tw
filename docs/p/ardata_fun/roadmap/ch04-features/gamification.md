## 11.1 為什麼「好玩」才是正確答案

做公民科技的人常常犯一個錯誤：把「重要性」當作人們參與的理由。

「這件事很重要，所以大家應該要幫忙。」

這個邏輯在現實中行不通。政治獻金資料固然重要，但「重要」並不能讓人在下班後打開電腦，花半小時去查一個陌生公司的捐款紀錄。人的注意力是稀缺的，競爭對手是 Netflix、社群媒體、朋友聚會。

看看成功的群眾外包案例：

**Galaxy Zoo**（2007年）：天文學家把兩百萬張星系照片放上網，請大家幫忙分類。他們設計了一個像遊戲的介面——點擊、分類、下一張——並加入即時回饋。六個月內有 15 萬名志願者分類了超過一百萬張照片，做到專業天文學家要花三年才能完成的工作。

**Foldit**（2008年）：蛋白質折疊是電腦模擬多年未能突破的科學問題。研究人員把它做成益智遊戲。玩家不需要懂生化，只需要「把這個形狀拼好看」。結果：一群遊戲玩家在三週內解決了困擾 AIDS 研究者十五年的蛋白質結構問題。

**g0v（零時政府）**：台灣的公民科技社群。黑客松不是義工活動，而是節慶。大家帶著好奇心、炫技心態、交朋友的慾望來參加，順便解決了一些公共問題。

這三個案例的共同點：**參與行為本身就是令人愉快的**。人們不是因為「這件事重要」才做，而是因為「這件事有趣、讓我感覺自己在做有意義的事、還能被別人看見」。

本系統的核心設計問題：**如何讓「查資料 + 填表單」變成令人欲罷不能的體驗？**

偵探敘事框架是我們的答案。當一個人可以說「我是首席偵探，已經破了三十個案子」，他已經不是在「幫忙做一件重要的事」，而是在「做一件屬於自己的事」。這才是讓群眾外包真正運作的力量。

## 11.2 偵探敘事框架（Detective Narrative Frame）

整個 UX 用偵探辦案比喻包裝：

| 元素 | 一般說法 | 偵探說法 |
|------|---------|---------|
| 使用者 | 訪客 | 公民偵探 |
| 每筆捐款資料 | 一筆記錄 | 一條線索 |
| 回報疑似對價關係 | 填寫表單 | 結案 |
| 成就/徽章 | 積分 | 偵探職階晉升 |
| 排行榜 | 統計 | 局長表彰 |

### 首次訪問 Onboarding 文案

```
歡迎，偵探。

台灣政治獻金是公開的，但沒有人有時間逐一翻閱。直到現在。

你的任務：在 28,869 家企業的捐款紀錄中，
找出金錢與決策之間的軌跡。

每一筆異常的捐款，都可能是一條線索。
每一個被忽視的時間巧合，都等待著你去發現。

準備好了嗎？
```

| 徽章ID | 名稱 | 觸發條件 | 圖示 | 文案 |

## 11.4 偵探職階系統

| 職階 | 條件 | 稱號 |
|------|------|------|
| 0 | 初次進入 | 見習偵探 |
| 1 | 查閱 5 筆資料 | 初級偵探 |
| 2 | 回報 1 筆 | 正式偵探 |
| 3 | 獲得 3 個徽章 | 資深偵探 |
| 4 | 回報 10 筆 | 首席偵探 |
| 5 | 獲得所有徽章 | 傳奇偵探 |

職階晉升時觸發全螢幕慶祝特效，展示新稱號與解鎖說明。

## 11.5 視覺獎勵機制

| 事件 | 視覺效果 |
|------|---------|
| 解鎖成就 | Toast 通知（右下角）+ canvas confetti 動畫（整個螢幕） |
| 職階晉升 | 全螢幕慶祝特效 + 音效（可靜音） |
| 大額捐款（>100萬） | 紅色脈衝 glow 動畫（CSS `@keyframes pulse`） |
| 首次打開捐贈者詳頁 | 「正在解密檔案...」滑入動畫 |
| 搜尋觸發時 | 「掃描資料庫中...」打字機效果 |

### Toast 通知格式

```
┌─────────────────────────────────────┐
│ 🏅  成就解鎖！                        │
│     資深偵探                          │
│     已獲得 3 個徽章                   │
└─────────────────────────────────────┘
```

右下角滑入，4 秒後自動消失。

### confetti 動畫

使用 canvas-confetti 套件（CDN 引入）。只在職階晉升時觸發，成就解鎖時只顯示 Toast，避免過度干擾。

## 11.6 社群分享機制

每次搜尋/查閱都生成可分享的 URL：

```
?q=統一編號&tab=business
?q=05637971
?tab=individual
```

查閱捐贈者詳頁時提供「分享此調查」按鈕，點擊後：
1. 複製含目標企業搜尋條件的 URL 到剪貼簿
2. 觸發 SHARER 成就
3. 顯示 Toast：「調查連結已複製，快分享給其他偵探！」

分享文案（可選，用於社群媒體）：

```
我剛追蹤了[公司名稱]跨越[N]屆選舉的政治獻金，
總計捐出 [金額] 元。
你也來查查看 → [URL]

#公民偵探 #政治獻金調查
```

## 11.7 進度感設計（Progress Visibility）

頁面頂部常駐顯示：

```
共 28,869 家企業 | 你查閱了 X 家 | 你的職階：[稱號]
```

進度條顯示「距離下一個職階還差 X 步」：

```
初級偵探 ━━━━━━━━━━░░░░░░ 正式偵探
          ↑ 你在這裡（已回報0/1筆）
```

「今日偵探報告」（可收合的面板）：

```
今日（2026-03-31）你查閱了：
・ 燿華電子股份有限公司
・ 大中福國企業有限公司
共 2 家企業
```

這份記錄存於 localStorage，隔天清零，形成每日回訪的習慣迴路。

## 11.8 localStorage Schema

成就資料存於 `localStorage` key：`ardata_dpp_achievements`

```javascript
{
  schema_version: 2,
  stats: {
    total_reports: 0,          // 總回報次數
    total_views: 0,            // 總查閱企業次數
    big_fish_found: 0,         // 發現 >100萬 的次數
    whale_found: 0,            // 發現 >1000萬 企業的次數
    first_report_at: null,     // 第一次回報的 ISO 時間戳
    last_active_date: null,    // 最近活躍日期（YYYY-MM-DD）
    consecutive_days: 0,       // 連續使用天數
    unique_donors_viewed: [],  // 查閱過的 tax_id 列表
    elections_viewed: [],      // 查閱過的選舉名稱集合
  },
  unlocked: {
    // "FIRST_LOOK": { unlocked_at: "2026-03-31T10:00:00+08:00" }
  },
  rank: 0,                     // 當前職階（0-5）
  today_viewed: [],            // 今日查閱企業名稱（隔天清零）
}
```
