# CSV 格式規格

本系統使用三種真實 CSV 格式，均來自 PHP Pipeline 的實際輸出。

---

## Format 1：原始 incomes.csv（ZIP 內）

來源：`data/individual/account/{選舉}/{地區}/{姓名}_{serial}.zip` 解壓後

| 索引 | 欄位名稱 | 範例值 | 說明 |
|------|---------|--------|------|
| [0] | 序號 | `1` | 流水號 |
| [1] | 擬參選人/政黨 | `林右昌` | 收受捐款者 |
| [2] | 選舉名稱 | `107年縣(市)長選舉` | 完整選舉名稱 |
| [3] | 申報序號/年度 | `1` | 申報期別 |
| [4] | 交易日期 | `1070921` | 民國格式，見注意事項 |
| [5] | 收支科目 | `營利事業捐贈收入` | 法定申報分類 |
| [6] | 捐贈者/支出對象 | `大中福國企業有限公司` | 公司或個人名稱 |
| [7] | 身分證/統一編號 | `28539099` | 法人為 8 碼統編 |
| [8] | 收入金額 | `50000.00` | 帶 .00 後綴 |
| [9] | 支出金額 | （空白） | 收入列為空 |
| [10] | 支出用途 | （空白） | 收入列為空 |
| [11] | 金錢類 | `支票` | 支票 / 現金 等 |

## Format 2：個別企業捐款明細（report/incomes/business/{統一編號}.csv）

有標頭行：

```
選舉,捐贈對象,捐贈人,捐贈日期,捐贈金額
```

| 欄位 | 範例值 | 說明 |
|------|--------|------|
| 選舉 | `105年立法委員選舉` | 選舉名稱 |
| 捐贈對象 | `丁守中` | 候選人姓名 |
| 捐贈人 | `燿華電子股份有限公司` | 企業名稱 |
| 捐贈日期 | `104/09/16` | 民國格式（斜線分隔） |
| 捐贈金額 | `500000` | 數字字串，可能含或不含 .00 |

範例完整列：
```
105年立法委員選舉,丁守中,燿華電子股份有限公司,104/09/16,500000
107年縣(市)長選舉,林右昌,大中福國企業有限公司,1070921,50000.00
```

---

## Format 3：企業捐款排行榜（report/incomes/business.csv）

無標頭。每行格式：`total_amount,tax_id`

| 欄位 | 範例值 | 說明 |
|------|--------|------|
| total_amount | `17250000` | 歷次捐款總金額（整數，無 .00） |
| tax_id | `05637971` | 企業統一編號（8碼） |

範例：
```
17250000,05637971
8500000,12345678
3000000,87654321
```

依 total_amount 降序排列（最大捐款者在最前面）。

## 解析注意事項

### 日期格式

欄位 [4] 有兩種民國日期格式並存：

- 無分隔符：`1070921` → 民國 107 年 09 月 21 日
- 有分隔符：`104/09/16` → 民國 104 年 09 月 16 日

JavaScript 轉換：
```javascript
// 無分隔符格式
const s = '1070921';
const year = parseInt(s.substring(0, 3)) + 1911;  // 2018
const month = parseInt(s.substring(3, 5)) - 1;    // 8 (0-based)
const day = parseInt(s.substring(5, 7));           // 21

// 有分隔符格式
const parts = '104/09/16'.split('/');
const year2 = parseInt(parts[0]) + 1911;           // 2015
```

### 金額解析

`parseFloat(amount)` 即可處理帶或不帶 `.00` 的金額字串。

### 編碼

所有 CSV 均為 UTF-8，無 BOM。PapaParse 預設可正確處理。
